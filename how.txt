# Implementation Notes — HOW to build the Trading Agent
# This file contains the "how" details. The requirements (what/why) are in trading_agent_req.txt.
# These are suggestions — Claude Code decides final implementation during planning.

================================================================================
## HOW: 7-Year Learning Phase — Sequential Day-by-Day Processing
================================================================================

Process each trading day in date order, simulating what a live system would do.

For each day:
  1. Load that day's 1-min data for all 500 stocks
  2. Run all 22 strategies using ONLY information available before that day
     (no look-ahead — do not use future candles to compute any indicator)
  3. Record: which stocks were recommended, entry/target/stop prices
  4. Record: actual outcome — did price hit target or stop first?
  5. Update each strategy's rolling performance window
  6. Every 20 trading days: recalculate adaptive weights
  7. Generate Jupyter notebook for that day
  8. At each year-end: write yearly summary to memory.md

The outcome is already known (historical data), so analysis and evaluation
happen simultaneously. No separate test needed within the 7-year period.
After 7 years: freeze weights → begin 3-year holdout test.

================================================================================
## HOW: Adaptive Strategy Weighting (Unlearning Mechanism)
================================================================================

Weights are recalculated every 20 trading days based on rolling performance.

Weight update rule (based on last 10 signals per strategy):

  Win rate > 60%  →  weight × 1.5  (boost)
  Win rate 40-60% →  weight × 1.0  (unchanged)
  Win rate 30-40% →  weight × 0.5  (reduce)
  Win rate < 30%  →  weight × 0.1  (near-suspend)

  Minimum floor: 0.1  (never fully removed)
  Maximum cap:   3.0  (no single strategy dominates)

Regime-based weight override (applied daily before composite scoring):
  High VIX (>20):        multiply all breakout strategy weights by 0.3
  Low VIX + ADX > 25:   multiply all reversion strategy weights by 0.5

Near-suspended strategy revival:
  If weight is at 0.1 AND its natural regime returns → reset to 0.5
  and allow it to earn back over next 20 trading days.

All weight changes logged to memory.md with date and reason.

================================================================================
## HOW: 3-Year Holdout Backtesting
================================================================================

- Apply the weights frozen after 7-year learning to the unseen 3-year period
- Walk-forward: generate recommendations one quarter at a time,
  rolling forward (train on 7yr, test Q1 of year 8, then train on 7yr+Q1,
  test Q2, etc.) — prevents overfitting to a single test period
- Each day in the 3-year test: generate recommendations BEFORE seeing outcome
- Record: recommended stocks, entry/target/stop
- Record: actual P&L outcome
- Compute all performance metrics at end

================================================================================
## HOW: Composite Scoring
================================================================================

For each stock on each day:
  composite_score = sum(strategy_signal[i] × current_weight[i]) for all i

  strategy_signal values:
    +1  = strategy says buy
     0  = no signal
    -1  = strategy says sell (if shorting enabled; currently cash equity long only)

  Weight for each strategy starts at 1.0 and evolves via the adaptive rules above.

Top 3 stocks by composite_score (must also pass all quality filters) = recommendations.

================================================================================
## HOW: Data Download (Kite Connect)
================================================================================

- Max 60 days per API request for minute data
- 500 stocks × ~61 chunks (3650/60) = ~30,500 API calls total
- Rate limit: ~3 requests/second → min 170 min continuous, realistically
  several days with retries
- Downloader must: chunk by 60-day windows, sleep between calls, track
  per-stock completion in progress.json, retry failed stocks with backoff
- Also download: Nifty 50 index (NIFTY 50), India VIX (INDIA VIX)
  as separate instruments via same API

================================================================================
## HOW: Distributed Agent Memory File Structures
================================================================================

Each agent has its own memory file. All files live in the memory/ folder.
The Manager Agent is the only agent that writes to global.md.
Every agent reads its own file + global.md before making any decision.

--- memory/news_agent.md ---

## NEWS EVENT IMPACT PATTERNS
- RBI rate hike announcement: Banking sector falls 1-2% in first 30 min;
  HDFC Bank, ICICI Bank, Axis Bank most affected; reversal often by 11 AM
- Crude oil spike >3%: ONGC, Reliance gap up; Aviation stocks (IndiGo) gap down
- US Fed hawkish surprise: IT sector (TCS, Infy) falls on USD-INR strength
- Earnings beat >10%: stock gaps up, continuation 70% of time on day of results
- Earnings miss >10%: stock gaps down, gap-fill rare on results day

## SECTOR SENSITIVITY MAP
- Rising USD: IT positive, FMCG negative
- Rising rates: Banks volatile, Real estate negative
- Global risk-off: Metals and PSU stocks sell off first

## CORPORATE CALENDAR PATTERNS
- Avoid trading stock 2 days before earnings announcement (high uncertainty)
- Ex-dividend dates: stock usually underperforms sector on that day

## RECOMMENDATION PERFORMANCE LOG
| Date  | Stock   | Strategy | Entry | Target | Stop | Outcome | P&L  |
|-------|---------|----------|-------|--------|------|---------|------|
...

## WHAT MAKES A WINNING RECOMMENDATION (learned patterns)
- ORB-15 calls work best on Monday and Tuesday (fresh weekly momentum)
- VWAP reversion calls fail when VIX > 18 (abandon this in volatile sessions)
- Stocks in top 20 relative strength in first 30 min have 65% win rate
- Never recommend a stock with earnings within 3 days

## TIME-OF-DAY PATTERNS
- Best entry window: 9:30-10:30 AM (post-opening volatility settles)
- Worst entry window: 11:30 AM-1:00 PM (lunch lull, low volume, choppy)
- 2:00-3:00 PM: institutional squaring — reversion trades work well

--- memory/critique_agent.md ---

## FAILURE MODE LIBRARY (conditions that killed past recommendations)
- ORB call failed: Nifty reversed direction at 10:15 AM → always check Nifty
  direction before acting on individual stock ORB signal
- VWAP reversion failed: stock had negative news that morning, fundamental
  override prevented technical reversion
- Gap continuation failed: gap was >4% on low volume — exhaustion gap, not
  continuation gap; volume must confirm
- CPR trade failed: overall market was in distribution mode — sector was weak

## RED FLAGS CHECKLIST (check each recommendation against these)
- Is Nifty itself at a key resistance level today?
- Does stock have earnings, results, or AGM this week?
- Is sector in distribution (falling relative to Nifty)?
- Has stock already moved >2% from open before our entry?
- Is ADX < 15 (too directionless for a breakout trade)?
- Is there a major global event tonight (Fed, US jobs data)?

## CURRENT MARKET REGIME
Updated daily: regime type, VIX level, dominant strategy class, weight overrides.

## AGENT DISAGREEMENT LOG
